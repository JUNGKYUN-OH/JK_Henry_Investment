# spec: ETF 투자 이력 관리 앱 (Henry Ledger)

## 개요

ETF 거래를 기록하면 종목별 평균단가·수익률·자산배분이 자동 계산되어 즉시 반영되는 개인 포트폴리오 관리 앱. 종목별로 총 투자금을 40일 분할 매수 계획으로 실행하고, 매일 갱신되는 목표 매도가(평균단가 × 1.10)를 확인할 수 있다.

## 범위

### 포함
- 티커 사전 등록·삭제 (거래 입력 시 목록에서 선택)
- 거래 입력·수정·삭제 (매수/매도, 날짜, 티커, 수량, 단가, 수수료)
- 포트폴리오 대시보드: 보유 중 종목 테이블 + 매도 완료 종목 별도 섹션
- 전체 포트폴리오 요약 (총 투자금, 총 평가금액, 총 수익률, 총 수수료)
- Yahoo Finance 현재가 조회 (버튼 클릭 시 on-demand, 실패 시 수동 입력 fallback)
- **분할 매수 계획**: 종목별 총 투자금을 40일 균등 분할, 일별 매수금액 자동 산출
- **일별 매수 기록**: 실제 매수가 입력 시 계획 누적 평균단가 자동 계산
- **목표 매도가**: 계획 평균단가 × 1.10을 매수 기록 즉시 갱신하여 표시
- USD 단일 통화

### 제외
- 로그인·인증 — 단일 사용자 앱, 불필요
- 차트·그래프 시각화 — MVP 이후 단계
- 증권사 자동 연동 — 수동 입력 지속성 먼저 검증
- 투자 의사결정 일지·메모 — 거래 기록 축적 후 추가
- 세금(양도소득세) 계산 — 정확성 요건 높음, 별도 범위
- 국내 ETF — 데이터 소스가 달라짐, 해외 ETF로 먼저 검증
- 멀티유저·공유 기능 — 단독 사용자
- 분할 횟수(40) 및 목표 수익률(10%) 사용자 변경 — 고정값으로 먼저 검증

## 시나리오

### 1. 티커 등록
- **Given** 티커 등록 목록이 비어 있는 상태
- **When** "SPY"를 입력하고 추가 버튼을 클릭
- **Then** 티커 목록에 "SPY"가 나타난다

성공 기준:
- [ ] "SPY" 입력 → 목록에 "SPY" 행이 추가된다
- [ ] 이미 등록된 "SPY"를 재입력 → "이미 등록된 티커입니다" 에러 메시지가 표시된다
- [ ] 빈 값으로 추가 시도 → 추가 버튼이 비활성화된다

### 2. 매수 거래 입력
- **Given** 티커 "SPY"가 등록되어 있고, 거래 목록이 비어 있는 상태
- **When** 거래 입력 폼에서 SPY, 매수, 수량 10, 단가 $450.00, 수수료 $2.50, 날짜 2024-01-15를 입력하고 저장
- **Then** 거래가 저장되고 대시보드의 보유 중 섹션에 SPY 행이 나타난다

성공 기준:
- [ ] 저장 후 거래 목록에 SPY 매수 1건이 표시된다
- [ ] 대시보드 SPY 행: 수량 10주, 평균단가 $450.00, 투자금액 $4,500.00 이 표시된다
- [ ] 포트폴리오 요약: 총 수수료 $2.50이 별도 항목으로 표시된다
- [ ] 필수 필드(티커, 매수/매도 구분, 수량, 단가, 날짜) 중 하나라도 비어 있을 때 저장 시도 → 해당 필드 아래에 에러 메시지가 표시된다
- [ ] 수량·단가에 0 이하 값 입력 시 → "0보다 큰 값을 입력하세요" 에러 메시지가 표시된다

**계획 연동 자동 채움 (활성 계획이 있는 티커 선택 시):**
- [ ] 활성 계획이 있는 티커(SPY) 선택 → 폼 상단에 "SPY 투자 계획 · 5일차 · 오늘 목표 $100.00" 컨텍스트 배너가 표시된다
- [ ] 최근 조회된 Yahoo Finance 현재가가 있는 경우 → 단가 필드에 마지막 조회가($473.50)가 자동 입력된다
- [ ] 단가가 자동 입력된 경우 → 수량 필드에 (일별 목표금액 ÷ 단가 = $100.00 ÷ $473.50 ≈ 0.211)가 자동 계산되어 입력된다
- [ ] 단가를 변경하면 수량이 (일별 목표금액 ÷ 새 단가)로 즉시 재계산된다
- [ ] 수량을 직접 수정하면 단가 변경 시 수량 자동 재계산이 해제된다 (이후 수량은 사용자가 직접 관리)
- [ ] 활성 계획이 없는 티커 선택 시 → 컨텍스트 배너 없음, 단가·수량 자동 채움 없음

### 3. 추가 매수 후 평균단가 재계산
- **Given** SPY 10주 @ $450.00 매수 기록이 있는 상태
- **When** SPY 10주 @ $430.00 매수를 추가 입력하고 저장
- **Then** 평균단가가 두 거래의 평균으로 재계산된다

성공 기준:
- [ ] 저장 후 SPY 수량: 20주, 평균단가: $440.00, 투자금액: $8,800.00 이 표시된다

### 4. 매도 거래 입력
- **Given** SPY 10주 @ $450.00 매수 기록이 있는 상태
- **When** SPY 5주 @ $480.00 매도를 입력하고 저장
- **Then** 보유 수량이 5주로 줄고 실현손익이 표시된다

성공 기준:
- [ ] 저장 후 SPY 보유 수량: 5주
- [ ] SPY 실현손익: +$150.00 표시 ((480 - 450) × 5)
- [ ] 보유 수량(5주)을 초과하는 수량(6주 이상) 매도 시도 → "보유 수량을 초과합니다" 에러 메시지가 표시된다

### 5. 전량 매도 → 매도 완료 섹션 이동
- **Given** SPY 10주 보유 중인 상태
- **When** SPY 10주 전량 매도를 입력하고 저장
- **Then** SPY가 "보유 중" 섹션에서 제거되고 "매도 완료" 섹션에 나타난다

성공 기준:
- [ ] 전량 매도 후 "보유 중" 테이블에 SPY 행이 없다
- [ ] "매도 완료" 섹션에 SPY 행이 나타나고 총 실현손익이 표시된다

### 6. 현재가 조회 성공
- **Given** SPY 10주 보유 중인 대시보드
- **When** "현재가 새로고침" 버튼 클릭
- **Then** Yahoo Finance에서 현재가를 조회하고 미실현손익이 계산되어 표시된다

성공 기준:
- [ ] 버튼 클릭 후 로딩 상태(버튼 비활성화 또는 스피너)가 표시된다
- [ ] 조회 성공 후 각 보유 종목에 현재가, 평가금액, 미실현손익(금액 및 %)이 표시된다
- [ ] 마지막 조회 성공 시각이 표시된다 (예: "최종 업데이트: 2024-01-15 14:32")

### 7. 현재가 조회 실패 → 수동 입력
- **Given** SPY 10주 보유 중인 대시보드
- **When** "현재가 새로고침" 버튼 클릭 후 API 호출 실패
- **Then** 에러 메시지와 수동 입력 폼이 나타난다

성공 기준:
- [ ] "현재가를 불러오지 못했습니다" 에러 메시지가 표시된다
- [ ] 보유 종목별 수동 현재가 입력 필드가 노출된다
- [ ] 수동으로 현재가 입력 후 확인 → 미실현손익이 계산되어 표시된다

### 8. 거래 수정
- **Given** SPY 매수 거래 (10주 @ $450.00)가 저장된 상태
- **When** 해당 거래를 수정 폼에서 단가를 $445.00으로 변경하고 저장
- **Then** 평균단가와 투자금액이 재계산되어 표시된다

성공 기준:
- [ ] 수정 폼에 기존 값(단가 $450.00 등)이 미리 채워져 있다
- [ ] 저장 후 SPY 평균단가: $445.00, 투자금액: $4,450.00 이 표시된다

### 9. 거래 삭제
- **Given** SPY 매수 거래 2건(10주 @ $450.00, 10주 @ $430.00)이 저장된 상태
- **When** 첫 번째 거래(10주 @ $450.00) 삭제 버튼 클릭 → 확인 다이얼로그에서 확인
- **Then** 해당 거래가 목록에서 제거되고 평균단가가 재계산된다

성공 기준:
- [ ] 삭제 버튼 클릭 시 확인 다이얼로그가 표시된다
- [ ] 확인 후 거래 목록에 1건만 남는다
- [ ] 삭제 후 SPY 평균단가: $430.00, 수량: 10주로 재계산된다
- [ ] 다이얼로그에서 취소 → 거래 목록이 변경되지 않는다

### 10. 티커 삭제
- **Given** "SPY" 티커가 등록되어 있는 상태
- **When** "SPY" 삭제 버튼 클릭
- **Then** 거래 기록 유무에 따라 다르게 처리된다

성공 기준:
- [ ] SPY에 거래 기록이 있을 때 삭제 시도 → "거래 기록이 있는 종목은 삭제할 수 없습니다" 에러 메시지가 표시된다
- [ ] SPY에 거래 기록이 없을 때 삭제 시도 → 확인 다이얼로그 후 목록에서 제거된다

---

### 11. 분할 매수 계획 생성
- **Given** SPY 티커가 등록되어 있고, SPY에 대한 활성 계획이 없는 상태
- **When** 총 투자금 $4,000, 티커 SPY로 계획 생성
- **Then** 일별 매수금액 $100 (= $4,000 ÷ 40)의 40일 계획이 생성되고 계획 목록에 나타난다

성공 기준:
- [ ] 생성 후 계획 목록에 SPY 계획이 나타나고 진행 상황 "0 / 40일" 이 표시된다
- [ ] 계획 상세에서 오늘의 매수 목표금액 $100.00, 잔여 투자금 $4,000.00이 표시된다
- [ ] SPY에 이미 활성 계획이 있는 상태에서 새 계획 생성 시도 → "해당 종목에 이미 진행 중인 계획이 있습니다" 에러 메시지가 표시된다
- [ ] 총 투자금 미입력 또는 0 이하 입력 시 → 해당 필드 아래에 에러 메시지가 표시된다

### 12. 1일차 매수 기록 및 목표 매도가 최초 확인
- **Given** SPY 계획(총 $4,000, 일별 $100)의 진행 상황이 "0 / 40일"인 상태
- **When** 1일차 매수로 SPY 0.212주 @ $472.00을 입력 후 저장
- **Then** 1일차 완료로 기록되고 목표 매도가가 최초 표시된다

성공 기준:
- [ ] 진행 상황: "1 / 40일"로 업데이트된다
- [ ] 잔여 투자금: $3,900.00으로 감소한다
- [ ] 계획 평균단가: $472.00, 목표 매도가: $519.20 (= $472.00 × 1.10)이 표시된다
- [ ] 거래 내역에 SPY 매수 1건이 추가된다
- [ ] 이미 오늘 매수 기록이 있는 날에 재기록 시도 → 확인 다이얼로그(중복 경고)가 표시된다

### 13. 이후 일차 매수 기록 시 목표 매도가 갱신
- **Given** SPY 계획에서 3일치 매수 완료 (계획 평균단가 $470.00), 목표 매도가 $517.00이 표시된 상태
- **When** 4일차 매수로 SPY @ $480.00 기록
- **Then** 계획 평균단가가 재계산되고 목표 매도가가 새 값으로 교체된다

성공 기준:
- [ ] 4일차 이후 계획 평균단가: $472.50 (= (470×3 + 480) ÷ 4, 근사값) 으로 업데이트된다
- [ ] 목표 매도가: 새 평균단가 × 1.10으로 교체 표시된다 (이전 값 $517.00은 사라진다)
- [ ] 진행 상황: "4 / 40일"로 업데이트된다

### 14. 계획 완료
- **Given** SPY 계획의 진행 상황이 "39 / 40일"인 상태
- **When** 40일차 매수를 기록
- **Then** 계획이 완료 상태가 되고 활성 계획 목록에서 제거된다

성공 기준:
- [ ] 진행 상황: "40 / 40일 완료"로 표시된다
- [ ] 최종 평균단가 및 최종 목표 매도가가 표시된다
- [ ] SPY로 새 계획을 생성할 수 있는 상태가 된다 (활성 계획 없음)

## 불변 규칙

**즉시 반영**: 거래를 추가·수정·삭제하면 평균단가·실현손익·포트폴리오 요약이 저장 즉시(페이지 새로고침 없이) 업데이트된다.

**수수료 분리**: 수수료는 평균단가 계산에 포함하지 않는다. 포트폴리오 요약에 총 수수료를 별도 항목으로 표시한다.

**평균단가법**: 같은 종목의 여러 매수 거래는 (총 매수금액 합계) ÷ (총 매수수량 합계)로 평균단가를 계산한다.

**현재가 조회 실패 시 기존값 유지**: API 조회 실패 시, 이전 조회에서 받아온 현재가와 조회 시각은 그대로 표시한다.

**계획 평균단가 독립**: 계획의 평균단가는 해당 계획에 속한 매수 기록만 집계한다. 계획 외 동일 종목 거래는 포함하지 않는다.

**계획 매수 → 거래 내역 자동 연동**: 계획에서 일별 매수를 기록하면 동일 내용이 거래 내역에도 매수 거래로 추가된다. 두 곳에 이중 입력하지 않는다.

**목표 매도가 갱신**: 매일 매수 기록 직후 계획의 목표 매도가(= 계획 평균단가 × 1.10)가 즉시 갱신된다. 이전 목표 매도가는 새 값으로 교체되어 하나의 값만 표시된다.
