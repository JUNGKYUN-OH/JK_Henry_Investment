# ETF 투자 이력 관리 앱 (Henry Ledger)

## Problem Statement

HMW — ETF 매수/매도를 기록할 때마다 엑셀 수식을 고치는 대신,
거래를 입력하면 평균단가·수익률·자산배분이 이미 계산되어 있는
상태를 항상 유지할 수 있을까?

## Recommended Direction

거래 입력(CRUD) + 자동 계산 + 현황 대시보드를 하나의 앱으로.

현재가는 Yahoo Finance 공개 API(별도 인증 불필요)로 해외 ETF 기준
조회하며, 사용자가 "새로고침" 버튼을 클릭할 때 조회한다. 계산 방식은
평균단가법(총 매수금액 ÷ 총 수량)을 사용한다. 수수료는 거래 입력 시
사용자가 직접 입력하고 별도 비용으로 처리한다(평균단가에 미포함).

핵심 계산: 평균단가, 실현손익, 미실현손익, 수익률.
대시보드: 보유 종목별 현황 테이블 + 자산배분 비율.

## Key Assumptions to Validate

- [ ] 수동 입력이 지속 가능하다 — 2주 동안 실제로 거래를 기록해보기
- [ ] Yahoo Finance API로 해외 ETF 현재가 조회가 안정적이다 — 무료 티어 rate limit 확인
- [ ] 엑셀보다 실제로 빠르다 — 거래 1건 입력 시간 비교

## MVP Scope

**포함:**
- 거래 입력 (매수/매도, 날짜, 수량, 단가, 수수료)
- 종목별 보유 현황 (평균단가, 수량, 평가금액, 수익률)
- 전체 포트폴리오 요약 (총 투자금, 평가금액, 총 수익률)
- Yahoo Finance API로 현재가 조회 (수동 입력 fallback 포함)

**제외:**
- 차트/그래프 시각화
- 증권사 자동 연동
- 매수 이유 메모 / 투자 일지
- 세금 계산 (양도세 등)
- 모바일 앱
- 국내 ETF 지원 (초기 범위 제외)

## Not Doing (and Why)

- **증권사 API 자동 연동** — 증권사별 정책이 복잡하고, 수동 입력 지속성이 먼저 검증되어야 함
- **투자 의사결정 일지** — Vitamin. 거래 기록이 쌓인 뒤에야 가치 발생
- **국내 ETF** — 데이터 소스가 달라지고 초기 복잡도가 높아짐; 해외 ETF로 먼저 검증
- **세금 계산** — 정확성 요건이 높고 MVP 범위 밖
- **공유/멀티유저** — 단독 사용자; 추가 복잡도 불필요

## Decisions Made

- **Yahoo Finance API**: 공개 API 사용 (인증 불필요, 무료)
- **현재가 갱신**: 사용자가 버튼 클릭 시 on-demand 조회
- **수수료 처리**: 거래 입력 시 사용자 직접 입력, 평균단가와 별도 집계
